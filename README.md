# get_2d_cell_from_pdf

Утилиты для получения цифровой сетки участков из PDF/PNG генплана.

Основной сценарий:

1. Конвертировать первую страницу PDF в `page_1.png`.
2. Распознать контуры участков на растре.
3. Получить превью, SVG-сетку и `polygon.json` в формате, совместимом с пайплайном `GetCoords/stage1_make_polygon.py`.

## Требования

- Python 3.9+
- Poppler для конвертации PDF через `pdf2image`
- Python-зависимости из `requirements.txt`

Установка зависимостей:

```bash
pip install -r requirements.txt
```

В `convert_pdf_to_png.py` должен быть указан путь к Poppler:

```python
POPLER_BIN = r"C:\tools\poppler-24.08.0\Library\bin"
```

## Быстрый запуск

Положите PDF генплана в папку проекта.

Сначала создайте растр:

```bash
python convert_pdf_to_png.py
```

Скрипт берёт первый найденный `*.pdf` в текущей папке и сохраняет первую страницу как:

```text
page_1.png
```

Затем запустите распознавание:

```bash
python detect_polygons_from_image.py
```

## Выходные файлы

`detect_polygons_from_image.py` создаёт:

| Файл | Назначение |
|---|---|
| `output_detected.png` | Цветное превью найденных участков |
| `polygon_grid.svg` | SVG-сетка контуров участков |
| `polygon.json` | Основной JSON в формате Stage 1 из `GetCoords` |
| `detected_polygons.json` | Сырой отладочный список полигонов в пикселях |

`polygon.json` имеет структуру:

```json
{
  "inc": 73,
  "data": [
    {
      "adres": "",
      "id": 1,
      "idtur": "00001",
      "kadastr": "",
      "kadastrurl": "",
      "names": "00001",
      "number": "1",
      "price": "",
      "size": "",
      "status": "sale",
      "coordinates": [
        [[y, x], [y, x]]
      ]
    }
  ]
}
```

Координаты центрируются относительно середины изображения. По умолчанию включён поворот осей `x <-> y`, чтобы структура была ближе к формату `stage1_make_polygon.py`.

## Что делает распознавание

`detect_polygons_from_image.py`:

- читает `page_1.png`;
- выделяет контуры участков через OpenCV;
- подавляет штриховку и серые заливки, чтобы они не склеивали участки;
- фильтрует крупные внешние оболочки и мелкие артефакты;
- схлопывает близкие вершины смежных участков;
- сохраняет превью, SVG и JSON.

Важные настройки находятся в верхней части файла:

```python
INPUT_IMAGE = "page_1.png"
OUTPUT_IMAGE = "output_detected.png"
OUTPUT_JSON = "polygon.json"
OUTPUT_SVG = "polygon_grid.svg"
PREVIEW_DPI = 600
SVG_STROKE_COLOR = "#ff0000"
SVG_LINE_WIDTH = 4
SNAP_TOLERANCE = 12
```

`SNAP_TOLERANCE` задаёт допуск схлопывания близких вершин между разными полигонами. Это помогает убрать зазоры между смежными участками.

## Ограничения текущего подхода

Текущий метод работает по растру. Контуры часто находятся по краю линии или заливки, а не по геометрической середине исходной границы.

Из-за этого:

- соседние участки могут иметь небольшой зазор до схлопывания вершин;
- наружные границы могут быть немного смещены относительно идеальной осевой линии;
- результат не является юридически точной кадастровой геометрией.

Этот инструмент подходит для восстановления визуальной схемы участков из генплана, но не заменяет координаты из официальных кадастровых источников.

## Normalizer

`normalize.py` оставлен как вспомогательный старый инструмент.

Он не распознаёт участки из изображения. Он берёт уже готовые полигоны из JSON, масштабирует/центрирует их относительно файла-образца и сохраняет нормализованный JSON.

Пример:

```bash
python normalize.py --input detected_polygons.json --schema polygon_sample.json --out polygon.normalized.json --auto
```

В текущем основном сценарии `normalize.py` обычно не нужен, потому что `detect_polygons_from_image.py` уже создаёт `polygon.json` в нужной структуре.

## Дальнейшее развитие

Следующий важный этап - переход от контуров по краю растровой линии к детекции по осевой линии границы.

Планируемое направление:

1. Выделять граничные линии участков как отдельную бинарную маску.
2. Удалять заливки, штриховки, подписи и декоративные элементы.
3. Выполнять скелетизацию линий, чтобы получить centerline толщиной в 1 пиксель.
4. Строить граф узлов и рёбер по этой осевой сетке.
5. Полигонализировать замкнутые области уже по осевым линиям.

Это должно дать более правильную топологию: смежные участки будут иметь одну общую границу, а не две параллельные границы по краям растровой линии.
